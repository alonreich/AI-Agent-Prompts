## 1. ABSOLUTE SYSTEM & OS CONSTRAINTS
### 1.1 Read/Write Permission Boundaries
* **The %PROGRAMFILES% Contract (READ-ONLY):** The NativeAOT executable, Avalonia DLLs, and bundled binaries (`ffmpeg.exe`, `ffprobe.exe`, `mpv.exe`, `libmpv-2.dll`) will reside here. Your code MUST NOT attempt to write to this directory.
* **The %PROGRAMDATA% Contract (WRITABLE):** All state transfers, IPC JSON configurations, logs, and temporary workspaces (`tempfile.gettempdir()`) must be explicitly routed to `%PROGRAMDATA%\Fortnite Video Software`.
* **Installer Logs:** Installation diagnostics must be written exclusively to `%TMP%\Fortnite_Video_Software_Install_Report.txt`.

### 1.2 IPC, Mutex Locking, & Self-Healing Configs
The suite runs multiple decoupled processes (Compressor, Merger, Crop Tool).
* **IPC via File System:** Processes communicate via `session_state.json` and `crops_coordinations.conf` located in `%PROGRAMDATA%`.
* **Global Mutex:** You MUST wrap all reads/writes to these shared JSON files in a Named System Mutex (e.g., `Global\FvsStateTransferMutex`) to prevent disk I/O race conditions during rapid process handoffs.
* **Cascading Backup Rotation:** Before writing to `crops_coordinations.conf`, the code MUST execute a 5-file atomic backup rotation (`.bak1` through `.bak5`). If a config is corrupted, it must self-heal using hardcoded defaults.

### 1.3 Crash Recovery Protocol (CRP)
You must implement a high-integrity CRP mirroring the original `RecoveryManager`:
* **Atomic Serialization:** State saves must be async and use a write-then-replace pattern (write to `.tmp`, then `File.Move` to overwrite).
* **Session Locking:** Maintain a PID-based session lock file (`app_session.lock`). Upon startup, if the file exists and the recorded PID is dead, trigger the crash recovery dialog.
* **Safe Mode Sentinel:** To prevent infinite crash loops, create a `safe_mode.sentinel` file. If the app crashes within 120 seconds of the sentinel's creation timestamp, bypass recovery and boot fresh.

## 2. NATIVE INTEROP & THREAD SAFETY (CRITICAL)
### 2.1 MPV P/Invoke & Deadlock Prevention
* **Direct P/Invoke:** You must use direct `[LibraryImport]` / P/Invoke bindings to `libmpv-2.dll`. Do NOT use heavy wrappers that might introduce marshaling overhead.
* **Unmanaged Delegate GC Pinning (CRITICAL):** You must explicitly mandate the use of `GCHandle.Alloc` (or `[UnmanagedCallersOnly]`) for any C# callback delegates passed to `libmpv-2.dll`. If unpinned, the .NET Garbage Collector will relocate or destroy the callbacks in memory, causing immediate native access violation (`0xC0000005`) hard crashes.
* **Spam-Click Resilience & Watchdog:** The app features multiple video preview screens. Users *will* aggressively spam click-to-seek buttons and scrub the timeline. The architecture must be entirely crash-proof against this. You MUST implement an `MPVSafetyManager` daemon thread. If an async seek operation to `libmpv` does not complete within 2.5 seconds, the watchdog must forcefully reset the internal seek state machine to unblock the UI.
* **Event Throttling:** Timeline scrubbing events firing to the MPV engine MUST be debounced/throttled to a maximum of 20fps (50ms intervals) to prevent native queue overflow.

### 2.2 FFmpeg Hardware Orchestration & Subprocess Safety
* **Subprocess Buffer Deadlocks (CRITICAL):** You must mandate asynchronous stream reading (`BeginOutputReadLine()` and `BeginErrorReadLine()`) for all `ffmpeg.exe` and `ffprobe.exe` executions. Synchronous reading or neglecting to flush the standard error/output buffers will cause the native OS pipe to fill up, permanently deadlocking the background worker threads.
* **The 15-Second HW Watchdog:** The background thread scanning for FFmpeg hardware encoders (`-hwaccels`) must be wrapped in a 15.0-second cancellation token timeout. If the GPU driver hangs, abort and fallback to CPU.
* **Strict Priority:** Execution order is immutable: `NVIDIA (h264_nvenc)` -> `AMD (h264_amf)` -> `INTEL (h264_qsv)` -> `CPU (libx264)`.
* **Encoder Contract:** All hardware encoders MUST use the `yuv420p` pixel format and H.264 `level 4.2` to guarantee cross-platform mobile playback.

## 3. TIMELINE ENGINEERING & MEDIA PIPELINE
### 3.1 Solid-Object Segment Rules
The granular speed and freeze frame timeline relies on strict "Solid Object" collision detection:
* **No Overlaps:** Segments CANNOT overlap. Dragging bounds must clamp against neighboring segments.
* **Minimum Duration:** Segment length is clamped to a minimum of `10ms`.
* **Freeze Frame Logic:** A freeze frame does NOT discard gameplay. It inserts a still image (looping the frame) and mathematically shifts all subsequent video/audio forward. The output file's total duration MUST mathematically increase by the exact duration of the freeze frame.

### 3.2 Portrait Canvas Trick & Coordinate Math
Do not compose directly at 1080x1920. Follow the scaling pipeline exactly:
1. **Backend Compose (1280x1920):** Source video is center-cropped here. Overlays are applied here to preserve HUD fidelity.
2. **Content Area (1080x1620):** The 1280x1920 composed canvas is scaled down to 1080x1620.
3. **Final Frame (1080x1920):** The 1080x1620 content is placed on a black 1080x1920 canvas at `Y=150`. This leaves the top 150px black strip reserved for text generation.
4. **Rounding Drift Protection:** When computing the inverse transforms, you MUST apply `+1px` to the LEFT edge for Stats/Health/Team regions, and `+1px` to the RIGHT edge for the Loot region to prevent bounding-box truncation.

### 3.3 Audio Processing
* **Ducking:** The game audio track MUST undergo a 150Hz frequency split (Highpass/Lowpass). The high frequencies are compressed (ducked) against the background music track using an RMS sidechain compressor (Threshold 0.15, Ratio 2.5).
* **Constant Tempo:** If the video speed is scaled (e.g., 2.0x), the background music track MUST remain at a constant `1.0x` tempo. Only the game audio scales.
* **WhatsApp Intro:** The first frame of the export MUST be prepended as a static lead-in image for exactly `<= 0.1 seconds`. This is a non-negotiable contract to bypass WhatsApp's black-screen thumbnail bug.
