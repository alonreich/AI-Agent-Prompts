- Hallucination risks
- Silent assumption risks

None.

If the item is Blocked, explain what blocks it.

### Acceptance Criteria
List clear pass/fail checks.
Each check must be testable.
Do not write vague checks.

Examples of valid acceptance criteria:
- The feature does X when Y happens.
- The system shows an error when required input is missing.
- The system does not overwrite existing content.
- The system does not invent missing data.
