KNOWN CAVEATS & SOLVED STRUGGLES (CRITICAL FOR THE NEXT AGENT):
1. The Deprecated `--agent` Flag: In OpenClaw 2026.6.5, the `chat`, `tui`, and `terminal` commands NO LONGER accept the `--agent <id>` flag. Using it will throw an "OpenClaw does not recognize option '--agent'" error. 
   - Solution: You MUST use the composite session key to target an agent. Instead of passing `--agent <id> --session main`, pass exactly: `--session agent:<id>:main`

2. Preventing Local Directory Pollution: OpenClaw will automatically generate files like `AGENTS.md`, `HEARTBEAT.md`, `IDENTITY.md`, and a `.openclaw/` folder inside whichever directory is specified as its workspace. 
   - Solution: When running `openclaw agents add` to create the isolated environment, do NOT point the `--workspace` parameter to the local project CWD. You MUST point it to the centralized profile directory (e.g., `%USERPROFILE%\.openclaw\sessions\[Hashed_CWD]\workspace`). This guarantees the user's project CWD remains completely pristine.

3. PowerShell `$pid` Crash: When writing the auto-termination logic to kill the Gateway process, do NOT loop using `foreach ($pid in $pids)`. `$pid` is a reserved, read-only automatic variable in PowerShell (it holds the current PowerShell process ID). Attempting to overwrite it will crash the wrapper script precisely when it tries to clean up the background processes.
   - Solution: Use an unreserved variable name, such as `foreach ($p in $pids)`.
